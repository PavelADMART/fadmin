# События
Для возможности более гибкой работы с настройками в «Конструкторе» предусмотрена система событий. Название всех событий определены в константах класса `\Rover\Fadmin\Options\Event`. Описание видов событий можно найти в [api по \Rover\Fadmin\Options\Event](./api/options/event.md#Константы)
 
Чтобы использовать событие в модуле, подключенном к «Констуктору», необходимо создать соответствующий метод в классе модуля, унаследованном от `\Rover\Fadmin\Options`. 

Если метод, определённый в константе, начинающейся с `BEFORE_`, вернет `false`, то действие, обычно совершаемое после этого события, выполнено не будет. Это не относится к событию `'beforeGetTabInfo'`. Оно позволяет только изменить информацию о вкладке, но не отменить ее отображение.

> На данный момент система событий основанаая на переопределении методов, считается устаревшей. В ближайших версиях появится новая, основанная на системе событий Битрикс d7.
